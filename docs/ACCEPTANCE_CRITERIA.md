# Acceptance Criteria

- Users can paste a supported Bilibili URL and inspect it from the frontend.
- The inspection response includes collection title, artwork, uploader, and episode list.
- Users can choose between quality presets before starting a download.
- Users can select or deselect episodes individually, or select all at once.
- Starting a download creates a job and shows progress in the UI.
- Completed items show the saved output path.
- The backend rejects unsupported URLs with a readable error.
- The UI remains usable on mobile widths.
- The product clearly signals that it is intended for authorized or publicly accessible content.
